# 社区治理与资源管理规范

## 1. 参考原则

- 参考 OpenKG：重视知识组织、来源透明、长期维护
- 参考 Hugging Face：重视数据集卡、模型卡、版本化与可复现

## 2. 任务管理规范

每个任务需具备：

- `task_id`（唯一标识）
- `description`（任务定义）
- `metric`（评测指标）
- `dataset`（数据来源）
- `baseline`（可运行基线）
- `owner_team`（维护人）

## 3. 数据集管理规范

每个数据集需具备：

- 名称、版本、语言、许可证、来源链接
- 训练/验证/测试切分定义
- 清洗或脱敏规则说明
- 数据质量说明与已知风险

## 4. 贡献流程

1. 使用 Issue 模板提出任务或数据需求  
2. Fork + 新建分支开发  
3. 通过 PR 模板提交，附评测结果与复现说明  
4. 维护者评审后合并，自动触发 CI 更新

## 5. 评测发布规范

- 所有分数必须标注评测日期和代码版本
- 关键基线更新必须附实验日志
- 排行榜应标记“官方基线”与“社区提交”
